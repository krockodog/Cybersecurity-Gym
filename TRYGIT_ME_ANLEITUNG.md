## 4. Quiz-Labor

### 4.1 Starten eines Quiz

1. Wahle **Quiz Labor** in der Navigation
2. Wahle deinen gewunschten Modus:
   - **Ubungs-Modus**: Unbegrenzte Zeit, sofortige Antwort-Feedback
   - **Prufungs-Modus**: Zeitbegrenzt, erst am Ende Feedback
3. Wahle die Anzahl der Fragen (5, 10, 20, 40 oder alle)
4. Klicke **Quiz starten**

### 4.2 Wahrend des Quiz

- **Frage-Karte**: Zeigt die aktuelle Frage mit allen Antwortoptionen
- **Fortschrittsbalken**: Visualisiert deinen Quiz-Fortschritt
- **Fragen-Raster**: Klicke auf eine Nummer, um direkt zu dieser Frage zu springen
- **80/20-Hinweis**: Klicke auf die Gluhbirne, um einen strategischen Hinweis zu erhalten
- **Sprachausgabe**: Lies die Frage und Antworten vor
- **Markieren**: Markiere Fragen zur spateren Uberprufung

### 4.3 80/20-Hinweis-System

Jede Frage hat einen kontextuellen Hinweis, der die **20% der Kenntnisse** hervorhebt, die **80% der Ergebnisse** bringen:

- Klicke auf die **Gluhbirne-Icon**
- Der Hinweis erscheint in einem Overlay
- Formuliert als Gedachtnisstutze, nicht als direkte Antwort

### 4.4 Quiz-Auswertung

Nach dem Quiz siehst du:
- **Ergebnis-Prozentsatz** (grun = bestanden >= 75%)
- **Richtig/Falsch-Aufschlusselung**
- **Zeit pro Frage**
- **Domain-Verteilung** der richtig/falsch beantworteten Fragen
- **Detaillierte Erklarung** fur jede Frage
- **Hinweis zur nachsten Lerneinheit**

### 4.5 Speicherung

Alle Quiz-Ergebnisse werden automatisch gespeichert und fliessen in:
- Deine Fortschrittsanalyse ein
- Die Schwachen-Erkennung des KI-Tutors
- Die XP-Berechnung

## 5. PBQ-Arena

### 5.1 Was sind PBQs?

**Performance-Based Questions (PBQs)** sind praktische Prufungsaufgaben, bei denen du:
- Netzwerk-Topologien analysierst
- Firewall-Regeln konfigurierst
- In Terminal-Simulationen Befehle eingibst
- Code auf Sicherheitslucken prufst
- Angriffs-Pfade nachvollziehst

### 5.2 PBQ-Typen

Die Arena enthalt **5 interaktive PBQ-Typen**:

| Typ | Beschreibung | Fertigkeiten |
|-----|-------------|--------------|
| **Netzwerk-Topologie** | Drag-and-Drop Netzwerk-Design | Subnetting, VLANs, Routing |
| **Firewall-Regeln** | Konfiguration von ACLs | Zugriffskontrolle, Port-Filter |
| **Terminal-Simulation** | Echte Linux/Windows-Befehle | CLI, Scripting, Tools |
| **Code-Review** | Sicherheitslucken finden | Secure Coding, CWE |
| **Angriffs-Pfad** | Schritt-folgen nachvollziehen | Kill Chain, MITRE ATT&CK |

### 5.3 Bedienung

1. Wahle einen PBQ-Typ aus der Ubersicht
2. Lies die Aufgabenstellung sorgfaltig
3. Fuhre die geforderten Aktionen durch
4. Klicke **Uberprufen**, um deine Losung zu validieren
5. Erhalte detailliertes Feedback mit Erklarung

## 6. 1-zu-1 KI-Tutor (Ollama)

### 6.1 Uberblick

Der **KI-Tutor** ist das Herzstuck der Plattform. Hier kannst du in Echtzeit mit einem KI-Professor sprechen, der:
- **Deine Schwachen** kennt (basierend auf Quiz-Ergebnissen)
- **Kontextbezogene Antworten** gibt (RAG-System)
- **Personalisierte Erklarungen** liefert
- **Streaming-Antworten** in Echtzeit zeigt

### 6.2 Starten einer Tutor-Sitzung

1. Offne **1-zu-1 Tutor** uber die Navigation
2. Wahle einen **Professor** aus dem Dropdown-Menu
3. Oder: Gehe uber Classroom direkt zu einem bestimmten Professor
4. Schreibe deine Frage in das Eingabefeld unten
5. Drucke **Enter** oder klicke **Senden**

### 6.3 Chat-Funktionen

- **Streaming-Antworten**: Die Antwort erscheint Wort fur Wort in Echtzeit
- **Code-Highlighting**: Code-Blöcke werden mit Syntax-Highlighting dargestellt
- **Markdown-Unterstutzung**: Tabellen, Listen, fetter/kursiver Text
- **TTS**: Klicke auf den Lautsprecher-Button, um Antworten vorlesen zu lassen
- **Chat-Verlauf**: Alle Chats werden pro Professor gespeichert
- **Vorschlage**: Schnell-Aktionen uber dem Eingabefeld ("Erklare...", "Ubungsfrage...")

### 6.4 Ollama-Verbindung

Der Tutor benotigt **Ollama** auf deinem Computer:

#### 6.4.1 Ollama Installation

**Windows:**
```powershell
# Lade Ollama von https://ollama.com/download herunter und installiere
# oder uber Winget:
winget install Ollama.Ollama
```

**macOS:**
```bash
# Lade Ollama von https://ollama.com/download herunter
# oder uber Homebrew:
brew install ollama
```

**Linux:**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

#### 6.4.2 Modell herunterladen

```bash
# Empfohlenes Modell (7B Parameter, schnell, gut fur Einsteiger):
ollama pull llama3

# Oder leistungsstarker (13B Parameter):
ollama pull llama3:13b

# Oder sehr leistungsstark (70B Parameter, benotigt mehr RAM):
ollama pull llama3:70b

# Alternative: Mistral (sehr effizient):
ollama pull mistral
```

#### 6.4.3 Ollama starten

```bash
# Starte den Ollama-Server (lauft automatisch im Hintergrund):
ollama serve

# Prufe ob Ollama lauft:
curl http://localhost:11434/api/tags
```

#### 6.4.4 Verbindungsstatus

Im Tutor siehst du oben links den **Verbindungs-Indikator**:
- 🟢 **Grun**: Verbindung zu Ollama hergestellt
- 🔴 **Rot**: Keine Verbindung - Ollama wird nicht gefunden

Wenn rot: Starte Ollama mit `ollama serve` und lade ein Modell mit `ollama pull llama3`

### 6.5 RAG-System (Retrieval-Augmented Generation)

Der Tutor verwendet **RAG** fur prazisere Antworten:

1. Deine Frage wird analysiert
2. Passende Fragen aus der Datenbank werden gesucht
3. Der Professor erhalt den **Kontext** dieser Fragen
4. Die Antwort basiert auf **echten Prufungsinhalten**

Dies bedeutet: Je mehr du ubt, desto besser wird der Tutor!

### 6.6 Schwachen-Analyse

Der Tutor kennt deine Schwachen:
- **Automatische Analyse** nach jedem Quiz
- **Schwachstellen-Panel** rechts im Tutor
- **Personalisierter Prompt**: Der Professor wird uber deine Schwachen informiert
- **Zielgerichtete Ubungen**: "Du hast Probleme mit SQL-Injection - hier ist eine gezielte Erklarung..."

## 7. Lernkarten mit Spaced Repetition

### 7.1 Prinzip

Lernkarten verwenden den **SM-2 Algorithmus** (wie Anki):
- Karten werden in **Intervalen** wiederholt
- Schwierige Karten erscheinen **haufiger**
- Einfache Karten erscheinen **seltener**
- Optimiert fur **langfristige Behaltensleistung**

### 7.2 Bedienung

1. Offne **Lernkarten** in der Navigation
2. Wahle ein **Deck** (z.B. "PenTest+ Domain 1: Planung")
3. Eine Karte wird angezeigt (Frage-Seite)
4. Uberlege dir die Antwort
5. Klicke **Umdrehen** oder auf die Karte
6. Die Antwort-Seite erscheint
7. Bewerte dein Wissen:
   - 🔴 **Wieder** (0) - Du wusstest es nicht
   - 🟠 **Schwer** (1-2) - Mit Muhe
   - 🟡 **Gut** (3) - Korrekt
   - 🟢 **Einfach** (4-5) - Sofort gewusst
8. Die nachste Karte erscheint automatisch

### 7.3 Deck-Struktur

Fur jede Zertifizierung gibt es mehrere Decks:
- **PenTest+**: 5 Decks (Planung, Informationssammlung, Angriffe, Nachbereitung, Tools)
- **Security+**: 6 Decks (Bedrohungen, Architektur, Implementierung, Operationen, Governance)
- **Network+**: 5 Decks (Netzwerk-Grundlagen, Infrastruktur, Netzwerk-Operationen, Sicherheit, Troubleshooting)
- **A+**: 4 Decks (Hardware, Betriebssysteme, Software-Troubleshooting, Sicherheit)
- **Linux LPI 1**: 4 Decks (Systemarchitektur, Installation, GNU-Befehle, Shell-Scripting)

### 7.4 Statistiken

Unten siehst du:
- **Heute fällig**: Anzahl der heute zu lernenden Karten
- **Neu**: Neue Karten im Deck
- **Gelernt**: Bereits beherrschte Karten
- **Wiederholen**: Karten die wiederholt werden müssen

## 14. Tipps fur maximale Erfolgsquote

### 14.1 Der 80/20-Ansatz

Konzentriere dich auf die **20% der Themen**, die **80% der Fragen** ausmachen:

**PenTest+:**
- Information Gathering (Tools: nmap, dig, whois)
- Web-Application-Angriffe (SQLi, XSS, CSRF)
- Netzwerk-Angriffe (MITM, ARP-Spoofing)
- Berichterstattung (CVSS, Risiko-Matrix)

**Security+:**
- Kryptografie (Symmetrisch/Asymmetrisch, Hashing)
- Bedrohungsakteure (APTs, Script Kiddies, Insider)
- Netzwerk-Sicherheit (Firewall, IDS/IPS, VPN)
- Identitatsmanagement (MFA, SSO, RADIUS)

**Network+:**
- OSI-Modell & TCP/IP (Alle 7 Schichten)
- Subnetting (CIDR, Berechnung)
- Routing-Protokolle (OSPF, BGP, EIGRP)
- Troubleshooting-Methodik

## 15. Fehlerbehebung

### 15.1 Plattform ladt nicht

| Problem | Losung |
|---------|--------|
| Weisse Seite | Browser-Cache leeren (Ctrl+Shift+R) |
| Lade-Fehler | URL prufen: `https://b4si6zbax4skg.kimi.page` |
| Netzwerk-Fehler | Internetverbindung prufen |

### 15.2 Ollama-Verbindung fehlschlagt

| Problem | Losung |
|---------|--------|
| Roter Punkt | `ollama serve` im Terminal gestartet? |
| Modell nicht gefunden | `ollama pull llama3` ausfuhren |
| CORS-Fehler | Ollama erlaubt lokale Verbindungen standardmassig |
| Langsame Antworten | Kleineres Modell probieren: `ollama pull llama3:7b` |

### 15.3 Sprachausgabe funktioniert nicht

| Problem | Losung |
|---------|--------|
| Kein Ton | Lautstärke prufen, Kopfhorer angeschlossen? |
| Stimme nicht auf Deutsch | Browser-Sprache auf Deutsch einstellen |
| Chrome funktioniert nicht | Firefox oder Edge probieren |
| Keine Stimme verfugbar | `speechSynthesis.getVoices()` im Browser-Konsole prufen |

### 15.4 Fortschritt nicht gespeichert

| Problem | Losung |
|---------|--------|
| Daten weg | localStorage wird verwendet - nicht im Privat-Modus lernen |
| Neuer Browser | Daten sind browserspezifisch |
| Speicher voll | Alte Browser-Daten loschen |

### 15.5 Allgemeine Tipps

- **Aktuellen Browser** verwenden (Chrome, Firefox, Edge)
- **JavaScript aktiviert** lassen (erforderlich)
- **Kein Ad-Blocker** auf der Seite (kann TTS blockieren)
- **Regular neu laden** bei Problemen: `Ctrl + Shift + R`
